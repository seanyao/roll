# Autonomous Operation — Domain Model

> Bounded Context: **Autonomous Operation**
> Engineering-layer documentation. Extracted from `bin/roll` and `skills/roll-loop`, `roll-.dream`, `roll-peer`. English only.

---

## Ubiquitous Language

| Term | Definition |
|---|---|
| **Story** | Smallest independently shippable work unit. Identified by `US-XXX`, `FIX-XXX`, or `REFACTOR-XXX`. Tracked in BACKLOG.md. |
| **BACKLOG** | Project-scoped ordered list of stories, bugs, and refactors. Single source of truth for what needs to be done. |
| **Loop cycle** | One execution pass of the autonomous executor: pick story → execute → verify → mark done. Bounded by LOCK. |
| **Active window** | Daily time range (`loop_active_start`–`loop_active_end`) during which loop is permitted to execute. |
| **LOCK** | Per-project file preventing concurrent loop execution. Created on cycle start, removed on exit (even on crash). |
| **In Progress** (🔨) | Story status indicating it is claimed by an executor (loop or human). |
| **TCR cycle** | Test && Commit \|\| Revert micro-step. Smallest shippable code change with a guaranteed green state. |
| **ALERT** | Signal written to `ALERT.md` when an automated check fails or an anomaly is detected. Requires human attention. |
| **Brief** | Human-readable digest of project state: completed stories, pending items, release readiness. Generated by `roll-brief`. |
| **Dream scan** | Nightly automated inspection of code health. Produces REFACTOR items for the BACKLOG. |
| **Peer session** | Cross-agent review exchange. Primary agent proposes; peer agent (different AI) responds with AGREE/REFINE/OBJECT/ESCALATE. |
| **Orphan 🔨** | Story stuck in `🔨 In Progress` because the loop that claimed it crashed before finishing. Detected and healed at cycle start. |
| **Run record** | Single JSONL line appended to `runs.jsonl` at the end of each loop cycle. Permanent audit trail. |

## Aggregates

### Loop Aggregate

**Aggregate Root**: `LoopCycle`

```
LoopCycle
  ├── id: run_id (loop-YYYYMMDD-HHMM)
  ├── status: LoopStatus (idle | running | paused | failed)
  ├── items: Story[]          — stories executed this cycle
  ├── started_at: timestamp
  ├── agent: AgentName
  └── lock: LockFile           — per-project LOCK; held for cycle lifetime

Story (entity within LoopCycle)
  ├── id: StoryId (US-XXX / FIX-XXX / REFACTOR-XXX)
  ├── status: StoryStatus (Todo | InProgress | Done)
  ├── started_at: timestamp
  └── tcr_count: int           — TCR micro-commits since start

LockFile (value object)
  ├── path: ~/.shared/roll/loop/.LOCK-<project-slug>
  └── pid: ProcessId

Alert (entity)
  ├── story_id: StoryId
  ├── reason: string
  ├── raised_at: timestamp
  └── path: ~/.shared/roll/loop/ALERT.md
```

**Invariants:**
- Only one `LoopCycle` with status `running` per project at a time (enforced by LOCK).
- A Story may only be picked up when its status is `📋 Todo`.
- A Story is not marked `✅ Done` until both TCR gate (tcr_count > 0) and CI gate pass.

**Commands:**

| Command | Precondition | Result |
|---|---|---|
| `StartCycle` | No LOCK for project | LoopCycle created, LOCK written, `CycleStarted` emitted |
| `PickStory` | Story.status = Todo | Story.status → InProgress, `StoryPickedUp` emitted |
| `CompleteStory` | tcr_count > 0, CI green | Story.status → Done, `StoryCompleted` emitted |
| `RevertStory` | TCR or CI gate failed | Story.status → Todo, `StoryReverted` + `AlertRaised` emitted |
| `PauseCycle` | LoopCycle running | status → paused, launchd unloaded |
| `ResumeCycle` | status = paused | launchd reloaded, status → idle |
| `HealOrphans` | Cycle start | Orphan 🔨 stories reverted to Todo, `AlertRaised` per orphan |

### Dream Aggregate

**Aggregate Root**: `DreamScan`

```
DreamScan
  ├── id: scan_id (dream-YYYYMMDD)
  ├── project: ProjectSlug
  ├── scanned_at: timestamp
  ├── observations: Observation[]
  └── refactor_items: RefactorItem[]

Observation (entity)
  ├── category: ObservationCategory (dead-code | duplication | naming | test-gap | ...)
  ├── location: FilePath
  └── description: string

RefactorItem (value object)
  ├── id: RefactorId (REFACTOR-NNN)
  ├── description: string
  └── source: "dream-YYYYMMDD"
```

**Invariants:**
- One DreamScan per project per night.
- RefactorItem.id must not collide with existing BACKLOG entries.

**Commands:**

| Command | Precondition | Result |
|---|---|---|
| `RunNightlyScan` | Scheduled 03:MM | DreamScan created, code health evaluated |
| `AppendRefactorItem` | Observation flagged | RefactorItem written to BACKLOG.md, `RefactorItemFlagged` emitted |
| `CommitScanReport` | Scan complete | Dream report committed to `docs/dream/` |

### Peer Aggregate

**Aggregate Root**: `PeerSession`

```
PeerSession
  ├── id: session_id
  ├── pair: AgentPair (primary:peer)
  ├── status: SessionStatus (active | resolved | abandoned | degraded)
  ├── rounds: ReviewRound[]
  └── resolution: Resolution?

ReviewRound (entity)
  ├── round_number: int
  ├── proposal: string      — primary agent's plan/diff
  ├── response: string      — peer agent's response
  └── verdict: Verdict (AGREE | REFINE | OBJECT | ESCALATE)

Resolution (value object)
  ├── verdict: Verdict
  ├── final_proposal: string
  └── resolved_at: timestamp
```

**Invariants:**
- ESCALATE verdict requires human decision before any code changes proceed.
- A PeerSession is abandoned if peer agent is unreachable after retry backoff.
- Maximum 3 review rounds per session before escalation.

**Commands:**

| Command | Precondition | Result |
|---|---|---|
| `StartPeerSession` | Plan affects >3 files or architecture decision | PeerSession created, `PeerSessionStarted` emitted |
| `SubmitRound` | Session active | ReviewRound appended, peer invoked |
| `ResolveSession` | Verdict received | Resolution set, `PeerSessionResolved` emitted |
| `AbandonSession` | Peer unreachable | status → abandoned, build continues without peer |

| Event | Published By | Consumed By |
|---|---|---|
| `CycleStarted` | Loop Aggregate | Observability (state.yaml updated) |
| `StoryPickedUp` | Loop Aggregate | Observability (BACKLOG 🔨 visible to brief) |
| `StoryCompleted` | Loop Aggregate | Observability (runs.jsonl appended, state.yaml idle) |
| `StoryReverted` | Loop Aggregate | Observability (BACKLOG → 📋 Todo) |
| `AlertRaised` | Loop Aggregate | Observability (ALERT.md written) |

| `PeerSessionResolved` | Peer Aggregate | Skill Delivery (build proceeds or pauses) |

## State Machine: Story Status

```
                    ┌─────────────────────────────────────┐
                    │                                     │
  initial ──→ [📋 Todo] ──→ [🔨 In Progress] ──→ [✅ Done]
                  ▲               │
                  │               │ TCR gate fails OR
                  └───────────────┘ CI gate fails OR
                    StoryReverted    orphan heal
```

Transitions:
- `📋 Todo → 🔨 In Progress`: loop picks story, commits "chore: mark US-XXX in progress"
- `🔨 In Progress → ✅ Done`: executor completes, TCR gate passes, CI gate passes
- `🔨 In Progress → 📋 Todo`: gate failure or orphan heal; ALERT raised

## Cross-Context Integration

**Loop → Observability** (via Published Language `runs.jsonl`):
After each cycle, `_loop_write_runs_jsonl` appends a record. Schema is strictly
enforced (FIX-018). `roll loop runs` reads this for human/agent visibility.

**Dream → Convention Management** (direct file write):
Dream scan appends REFACTOR-NNN rows directly to BACKLOG.md. BACKLOG.md is the
shared data owned by Convention Management but written by multiple contexts under
the "single file, well-known format" PL contract.

**Loop → Skill Delivery** (via skill invocation):
Loop calls `_agent_run_skill skills/roll-build/SKILL.md US-XXX`. Skill Delivery
executes the build in the appropriate AI agent. Loop does not know or care which
agent is used — that is Skill Delivery's concern.

**Peer → Skill Delivery** (gating):
A peer session blocks the build (Skill Delivery) until resolution is AGREE or
REFINE-accepted. ESCALATE returns control to human; OBJECT triggers a plan revision.
