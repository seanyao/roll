```bash

cd cybernetix
bash install.sh
```

The script handles everything: execute permissions, `~/.local/bin` creation, PATH configuration, and first-time setup.

## Convention Management

Unified management of behavioral conventions for Claude Code / Kimi Code / Gemini CLI / Codex / Cursor.

### Commands

| Command | Use Case |
|---------|----------|
| `cybernetix setup` | First-time initialization of `~/.cybernetix/`, link skills to AI tools |
| `cybernetix sync` | Sync conventions to AI tool config paths |
| `cybernetix sync skills` | Refresh skills and repair symlinks |
| `cybernetix sync all` | Sync both conventions and skills |
| `cybernetix init [dir] [type] [tools]` | New project: generate convention files; existing project: auto-refresh |
| `cybernetix reset` | Reset local cache from repo source, force-sync all |
| `cybernetix status` | View current state, sync status, and skill links |

### Steps

```bash
# 1. New machine — one-time setup
cybernetix setup

# 2. New project — generate convention files
cybernetix init . fullstack claude       # Fullstack project, Claude only
cybernetix init my-app cli claude,cursor # CLI project, Claude + Cursor
cybernetix init .                        # Interactive selection

# 3. Refresh existing project (re-merge from latest templates)
cybernetix init .                        # Auto-detects type, no args needed
cybernetix init ~/projects               # Batch refresh all sub-projects

# 4. Edit global conventions, then sync
vim ~/.cybernetix/conventions/global/AGENTS.md
cybernetix sync

# 5. Broke something? Reset
cybernetix reset
```

### Layering

```
Global  ~/.claude/CLAUDE.md            ← managed by cybernetix sync
  ↓  auto-stacked
Project <project>/AGENTS.md            ← generated by cybernetix init
Project <project>/.claude/CLAUDE.md
```

Global conventions take effect automatically. Existing projects need no extra setup.

### Directory Structure

```
~/.cybernetix/
├── config.yaml                  # sync path configuration
└── conventions/
    ├── global/                  # global conventions (single source of truth)
    │   ├── AGENTS.md            # master document
    │   ├── CLAUDE.md / GEMINI.md / .cursor-rules
    └── templates/               # project type templates
        ├── fullstack/
        ├── frontend-only/
        ├── backend-service/
        └── cli/
```

## Skill System

### Workflow

```
Design → Build → Check → Fix → Loop
```

### Quick Reference

| Scenario | Command |
|----------|---------|
| Uncertain approach, need discussion | `$cnx-design "topic"` |
| One-sentence request | `$cnx-roll-build "add a ..."` |
| Execute an existing Story | `$cnx-story-build US-001` |
| Bug fix / small change | `$cnx-fix-build FIX-001` |
| High-risk logic (payments/auth) | `$cnx-spar "feature description"` |
| Deep research (product/company/tech) | `$cnx-research "subject"` |
| Initialize new project | `$cnx-init` |
| Patrol production | `$cnx-sentinel patrol` |
| Debug a page | `$cnx-bb-debug <URL>` |

### Steps

```bash
# New project (conventions + full scaffold in one step)
$cnx-init my-app

# Legacy project (detects existing structure, asks: merge / keep / overwrite)
$cnx-init

# Day-to-day development
$cnx-design "user login feature"         # Plan → produce BACKLOG Stories
$cnx-story-build US-001                  # Develop → TCR → CI → Deploy
$cnx-fix-build FIX-001                   # Fix
$cnx-roll-build "add search to admin"    # One-sentence → auto-split → deliver
```

### Full Skill List

| Skill | Phase | Function |
|-------|-------|----------|
| `$cnx-design` | DESIGN | Discuss approaches + design architecture + plan Stories |
| `$cnx-story-build` | BUILD | Execute Story (with parallel dispatch) |
| `$cnx-spar` | BUILD | Adversarial TDD |
| `$cnx-fix-build` | BUILD/FIX | Bug fix |
| `$cnx-roll-build` | DESIGN+BUILD | One-sentence quick implementation |
| `$cnx-sentinel` | CHECK | Scheduled patrol |
| `$cnx-bb-debug` | CHECK | Deep page diagnostics |
| `$cnx-bb-analyzer` | CHECK | Diagnostic report analysis |
| `$cnx-.code-review` | Support | Pre-commit code self-review |
| `$cnx-.qa-cover` | Support | Testing standards reference |
| `$cnx-.changelog` | Support | Auto-generate CHANGELOG |
| `$cnx-.echo` | Support | Passive intent clarification — restate and confirm before acting |
| `$cnx-research` | RESEARCH | Deep research with HV Analysis, PDF report output |
| `$cnx-init` | - | Project scaffold initialization |
