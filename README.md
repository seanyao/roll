## What is Roll?

Roll solves a specific problem: when developers on the same team use different AI clients (Claude Code, Cursor, Gemini CLI, Kimi, Codex, DeepSeek, Pi, OpenCode, Trae), each agent operates under different engineering constraints. One developer's Claude runs tests another's Cursor never even considers.

Roll fixes this through three mechanisms:

1. **Convention CLI** вҖ” a single source of engineering conventions distributed to every AI client on the machine (`~/.claude/`, `~/.cursor/`, `~/.gemini/`, `~/.kimi/`, `~/.codex/`, `~/.deepseek/`, etc.)
2. **Skill System** вҖ” 20 proven engineering practices (TDD, TCR, SRE, INVEST, DDD) encoded as instructions that any AI agent can follow
3. **Autonomous Evolution** вҖ” an optional layer that lets the agent work unattended: `roll-loop` executes BACKLOG items hourly, `roll-.dream` scans code health nightly, and `roll-brief` briefs the human each morning. The human retains sole authority over releases.

The result: any agent, any client, same constraints вҖ” and optionally, continuous autonomous delivery.

## Convention Management

Unified behavioral conventions for Claude Code / Gemini CLI / Cursor / Kimi / Codex / DeepSeek / Pi / OpenCode / Trae вҖ” all from one source.

### Commands

Commands fall into two categories: **bash commands** run pure shell logic; **agent commands** (marked with рҹӨ–) launch a full AI agent session to execute a SKILL.md.

| Command | Description |
|---------|-------------|
| `roll setup [-f]` | First-time install on this machine, or re-sync (use `--force` to overwrite local cache) |
| `roll update` | One-step upgrade: `npm install -g @seanyao/roll@latest` + re-sync via setup |
| `roll init` | New project: create `AGENTS.md` + `BACKLOG.md` + `docs/features/` |
| `roll status` | Show sync state, skill links, and detected AI tools |
| `roll backlog` | Show all pending tasks from BACKLOG.md |
| `roll agent [use <name>\|list]` | Per-project agent selection вҖ” affects all рҹӨ– commands |
| `roll loop <on\|off\|now\|status\|monitor\|resume\|reset>` | рҹӨ– Manage the autonomous BACKLOG executor (three-service: loop/dream/brief) |
| `roll brief` | рҹӨ– Show latest owner brief (regenerate if stale >24h) |
| `roll peer` | рҹӨ– Cross-agent code review and negotiation |
| `roll release` | рҹӨ– Sync changelog + version bump + tag + npm publish + GitHub Release |
| `roll` _(no args, in project dir)_ | Dashboard: loop status, pending count, latest brief summary |

### Typical Flow

```bash
# 1. Install on this machine
npm install -g @seanyao/roll
roll setup

# 2. Initialize a project (run from project root)
cd my-app
roll init

# 3. Upgrade to a new release
roll update

# 4. Enable autonomous evolution (optional)
roll loop on                  # three services: loop (hourly) + dream (nightly) + brief (daily)
roll loop monitor             # real-time top-like dashboard
roll brief                    # read the latest digest

# 5. Switch agent per project
roll agent use kimi           # all рҹӨ– commands now use kimi for this project
```

### How Convention Layering Works

```
Global  ~/.claude/CLAUDE.md         вҶҗ user-owned; roll setup appends @roll.md
        ~/.claude/roll.md            вҶҗ Roll conventions (written by roll setup/sync)
  вҶ“  auto-stacked
Project <project>/AGENTS.md         вҶҗ generated by roll init
        <project>/.claude/CLAUDE.md
```

Global conventions are additive and never overwrite existing files. Project conventions are injected per project via `roll init`.

### Directory Layout

```
~/.roll/
в”ңв”Җв”Җ config.yaml                  # sync path configuration
в””в”Җв”Җ conventions/
    в”ңв”Җв”Җ global/                  # single source of truth
    в”Ӯ   в”ңв”Җв”Җ AGENTS.md
    в”Ӯ   в”ңв”Җв”Җ CLAUDE.md / GEMINI.md / .cursor-rules
    в””в”Җв”Җ templates/               # project type templates
        в”ңв”Җв”Җ fullstack/
        в”ңв”Җв”Җ frontend-only/
        в”ңв”Җв”Җ backend-service/
        в””в”Җв”Җ cli/
```

## Skill System

Skills are instructions that encode proven engineering practices into a form AI agents can reliably follow. They live in `~/.roll/skills/` and are symlinked into each AI client's skill directory on `roll setup`.

### Workflow

```
Research вҶ’ Design вҶ’ Build вҶ’ Check вҶ’ Fix вҶ’ (loop)
```

### Quick Reference

| Scenario | Skill |
|----------|-------|
| Uncertain approach, need to think it through | `$roll-design "topic"` |
| Execute a planned Story | `$roll-build US-001` |
| Free-form feature request | `$roll-build "add search to admin"` |
| Bug fix | `$roll-fix FIX-001` |
| Fast backlog capture (bug / idea) | `$roll-idea "description"` |
| High-risk logic (payments, auth, state machines) | `$roll-spar "feature"` |
| Deep research (product / company / tech) | `$roll-research "subject"` |
| Cross-agent code review | `$roll-peer` |
| Patrol production for regressions | `$roll-sentinel patrol` |
| Debug a broken page | `$roll-debug <URL>` |
| Record a dev moment / diary entry | `$roll-notes "just fixed that nasty bug"` |
| Let the agent work overnight | `roll loop on` |

### Full Skill List

**Loop A вҖ” Research & Design**

| Skill | Description |
|-------|-------------|
| `$roll-research` | HV Analysis (Horizontal-Vertical) deep research framework, outputs PDF report |
| `$roll-design` | Multi-turn discuss вҶ’ [peer review] вҶ’ DDD model вҶ’ solution design вҶ’ [peer review] вҶ’ write Stories to BACKLOG |

**Loop B вҖ” Implementation & Iteration**

| Skill | Description |

| Skill | Description |

## Autonomous Evolution

Off by default. Enable with `roll loop on` to let the agent work unattended on your project.

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Base layer (always active)                             в”Ӯ
в”Ӯ  $roll-design вҶ’ $roll-build вҶ’ $roll-fix вҶ’ $roll-spar   в”Ӯ
в”Ӯ  Human drives every action                              в”Ӯ
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ  Autonomous layer (opt-in: roll loop on)                в”Ӯ
в”Ӯ  roll-loop   вҖ” hourly BACKLOG executor                  в”Ӯ
в”Ӯ  roll-.dream вҖ” nightly code health scan                 в”Ӯ
в”Ӯ  roll-brief  вҖ” daily morning digest + release readiness в”Ӯ
в”Ӯ  Human reviews briefs; retains sole release authority   в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

Three services are scheduled via macOS launchd (Linux: crontab) and managed as a unit:

- **roll-loop** runs hourly, picks up `рҹ“Ӣ Todo` items from BACKLOG, routes them (`US-XXX вҶ’ $roll-build`, `FIX-XXX вҶ’ $roll-fix`, `REFACTOR-XXX вҶ’ $roll-build`), and enforces TCR discipline вҖ” if a story completes with zero `tcr:` micro-commits, it reverts to Todo with an ALERT.
- **roll-.dream** runs nightly, scanning for dead code, architectural drift, and refactoring opportunities. Outputs `REFACTOR-XXX` entries to BACKLOG.
- **roll-brief** runs each morning (or on-demand via `roll brief`), producing an owner-facing digest with a release-readiness verdict.

The autonomous layer **never** invokes `roll-release`. Production deployment is always a human decision.

Use `roll loop monitor` for a real-time `top`-like view of all three services: launchd status, current execution state, pending queue, alerts, and live log tail.

Per-project agent configuration: `roll agent use <name>` writes `.roll.yaml` so each project can use a different AI client (claude / kimi / deepseek / pi / codex / opencode). Lookup order: `.roll.yaml` вҶ’ `~/.roll/config.yaml` вҶ’ `claude` (default).
