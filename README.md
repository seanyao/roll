## The Idea

Roll Рђћ named after Wukong (ТѓЪуЕ║), the legendary trickster, bound by a golden headband (_жЄЉу«Ї_). Full power, enforced discipline. That's the metaphor: AI agents are capable without constraint, but only consistent at team scale when their behavior is anchored by shared engineering standards.

The specific problem: in teams where developers use different AI clients (Claude Code, Cursor, Gemini CLI, Codex), each agent receives different engineering constraints. One developer's Claude passes tests another's Cursor never even runs. Roll makes **the constraints the constant**, not the model.

It does this through two mechanisms: a CLI that distributes a single source of conventions to every AI client on the machine, and a set of Skills that encode proven engineering practices (TDD, TCR, SRE, INVEST) as instructions an AI Agent can actually follow.

## Convention Management

Unified management of behavioral conventions for Claude Code / Kimi Code / Gemini CLI / Codex / Cursor.

### Commands

| Command | Use Case |
|---------|----------|
| `roll setup` | First-time setup: pull local cache + link skills (non-invasive) |
| `roll sync conventions` | Opt-in: add WK conventions via `@include` (never overwrites existing files) |
| `roll sync skills` | Refresh skills and repair per-skill symlinks |
| `roll sync all` | Sync both conventions and skills |
| `roll hooks install` | Opt-in: install global git hook for AI client auto-detection |
| `roll init [type]` | New project: generate convention files (cwd); existing: auto-refresh |
| `roll reset` | Reset local cache from repo source, then force-sync |
| `roll status` | View current state, sync status, and skill links |

### Steps

```bash
# 1. Install (one command Рђћ non-invasive, zero decisions required)
bash install.sh
# Рєњ CLI linked to PATH, wk-* skills linked into ~/.claude/skills/ etc.

# 2. New project Рђћ generate convention files (run from project root)
cd my-app
roll init fullstack    # Fullstack project, select tools interactively
roll init cli          # CLI project
roll init              # Interactive selection

# 3. Refresh existing project (re-merge from latest templates)
roll init              # Auto-detects type, no args needed

# 4. Optional: add WK global conventions (appends @roll.md, never overwrites)
roll sync conventions

# 5. Optional: install global git hook for AI client tagging
roll hooks install

# 6. Broke something? Reset
roll reset
```

### Layering

```
Global  ~/.claude/CLAUDE.md            Рєљ user-owned; @roll.md appended (opt-in)
        ~/.claude/roll.md                Рєљ WK conventions (written by roll sync)
  РєЊ  auto-stacked
Project <project>/AGENTS.md            Рєљ generated by roll init
Project <project>/.claude/CLAUDE.md
```

Project-level conventions are injected per project via `roll init`. Global conventions are opt-in and never overwrite existing files.

## Skill System

### Workflow

```
Design Рєњ Build Рєњ Check Рєњ Fix Рєњ Loop
```

### Quick Reference

| Scenario | Command |
|----------|---------|
| Uncertain approach, need discussion | `$roll-design "topic"` |
| One-sentence request | `$roll-fly-build "add a ..."` |
| Execute an existing Story | `$roll-story-build US-001` |
| Bug fix / small change | `$roll-fix-build FIX-001` |
| High-risk logic (payments/auth) | `$roll-spar "feature description"` |
| Deep research (product/company/tech) | `$roll-research "subject"` |
| Initialize new project | `$roll-init` |
| Patrol production | `$roll-sentinel patrol` |
| Debug a page | `$roll-bb-debug <URL>` |

### Steps

```bash
# New project (conventions + full scaffold in one step)
$roll-init

# Legacy project (detects existing structure, asks: merge / keep / overwrite)
$roll-init

# Day-to-day development
$roll-design "user login feature"         # Plan Рєњ produce BACKLOG Stories
$roll-story-build US-001                  # Develop Рєњ TCR Рєњ CI Рєњ Deploy
$roll-fix-build FIX-001                   # Fix
$roll-fly-build "add search to admin"    # One-sentence Рєњ auto-split Рєњ deliver
```
