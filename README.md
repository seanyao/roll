## The Idea

Roll makes **the constraints the constant**, not the model. In teams where developers use different AI clients (Claude Code, Cursor, Gemini CLI, Codex), each agent receives different engineering constraints. One developer's Claude passes tests another's Cursor never even runs.

It does this through two mechanisms: a CLI that distributes a single source of conventions to every AI client on the machine, and a set of Skills that encode proven engineering practices (TDD, TCR, SRE, INVEST) as instructions an AI Agent can actually follow.

## Convention Management

Unified management of behavioral conventions for Claude Code / Kimi Code / Gemini CLI / Codex / Cursor.

### Commands

| Command | Use Case |
|---------|----------|
| `roll setup` | First-time setup: pull local cache + link skills (non-invasive) |
| `roll sync conventions` | Opt-in: add Roll conventions via `@include` (never overwrites existing files) |
| `roll sync skills` | Refresh skills and repair per-skill symlinks |
| `roll sync all` | Sync both conventions and skills |
| `roll hooks install` | Opt-in: install global git hook for AI client auto-detection |
| `roll init [type]` | New project: generate convention files (cwd); existing: auto-refresh |
| `roll reset` | Reset local cache from repo source, then force-sync |
| `roll status` | View current state, sync status, and skill links |

### Steps

```bash
# 1. Install (one command ‚ÄĒ non-invasive, zero decisions required)
bash install.sh
# ‚Üí CLI linked to PATH, roll-* skills linked into ~/.claude/skills/ etc.

# 2. New project ‚ÄĒ generate convention files (run from project root)
cd my-app
roll init fullstack    # Fullstack project, select tools interactively
roll init cli          # CLI project
roll init              # Interactive selection

# 3. Refresh existing project (re-merge from latest templates)
roll init              # Auto-detects type, no args needed

# 4. Optional: add Roll global conventions (appends @roll.md, never overwrites)
roll sync conventions

# 5. Optional: install global git hook for AI client tagging
roll hooks install

# 6. Broke something? Reset
roll reset
```

### Layering

```
Global  ~/.claude/CLAUDE.md            ‚Üź user-owned; @roll.md appended (opt-in)
        ~/.claude/roll.md                ‚Üź Roll conventions (written by roll sync)
  ‚Üď  auto-stacked
Project <project>/AGENTS.md            ‚Üź generated by roll init
Project <project>/.claude/CLAUDE.md
```

Project-level conventions are injected per project via `roll init`. Global conventions are opt-in and never overwrite existing files.

## Skill System

### Workflow

```
Design ‚Üí Build ‚Üí Check ‚Üí Fix ‚Üí Loop
```

### Quick Reference

| Scenario | Command |
|----------|---------|
| Uncertain approach, need discussion | `$roll-design "topic"` |
| One-sentence request | `$roll-fly-build "add a ..."` |
| Execute an existing Story | `$roll-story-build US-001` |
| Bug fix / small change | `$roll-fix-build FIX-001` |
| High-risk logic (payments/auth) | `$roll-spar "feature description"` |
| Deep research (product/company/tech) | `$roll-research "subject"` |
| Initialize new project | `$roll-init` |
| Patrol production | `$roll-sentinel patrol` |
| Debug a page | `$roll-bb-debug <URL>` |

### Steps

```bash
# New project (conventions + full scaffold in one step)
$roll-init

# Legacy project (detects existing structure, asks: merge / keep / overwrite)
$roll-init

# Day-to-day development
$roll-design "user login feature"         # Plan ‚Üí produce BACKLOG Stories
$roll-story-build US-001                  # Develop ‚Üí TCR ‚Üí CI ‚Üí Deploy
$roll-fix-build FIX-001                   # Fix
$roll-fly-build "add search to admin"    # One-sentence ‚Üí auto-split ‚Üí deliver
```

### Full Skill List

| Skill | Phase | Function |
|-------|-------|----------|
| `$roll-design` | DESIGN | Discuss approaches + design architecture + plan Stories |
| `$roll-story-build` | BUILD | Execute Story (with parallel dispatch) |
| `$roll-spar` | BUILD | Adversarial TDD |
| `$roll-fix-build` | BUILD/FIX | Bug fix |
| `$roll-fly-build` | DESIGN+BUILD | One-sentence quick implementation |
| `$roll-sentinel` | CHECK | Scheduled patrol |
| `$roll-bb-debug` | CHECK | Deep page diagnostics |
| `$roll-bb-analyzer` | CHECK | Diagnostic report analysis |
| `$roll-.code-review` | Support | Pre-commit code self-review |
| `$roll-.qa-cover` | Support | Testing standards reference |
| `$roll-.changelog` | Support | Auto-generate CHANGELOG |
| `$roll-.echo` | Support | Passive intent clarification ‚ÄĒ restate and confirm before acting |
| `$roll-research` | RESEARCH | Deep research with HV Analysis, PDF report output |
| `$roll-init` | - | Project scaffold initialization |
