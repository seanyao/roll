# Plan: Legacy Documentation Automation (IDEA-012)

## Why

Legacy projects accumulate two kinds of documentation debt:

1. **Scattered docs** — READMEs in subdirectories, convention files buried in source, design
   notes that never made it to `docs/`. Developers spend time hunting instead of reading.

2. **Invisible conventions** — Architecture decisions, naming rules, environment variables, and
   module boundaries live only in code or developer memory. When they drift, there's no signal.

Human-only maintenance of these gaps fails silently: reminders are forgotten, PRs don't include
doc updates, and the codebase gradually diverges from its own documentation.

## Approach

Two deliverables — one new skill, one dream enhancement:

```
Story 1: roll-doc  (new skill, US-SKILL-008)
  Unified — "scan, index, identify gaps, fill gaps" in one invocation
  User calls $roll-doc; skill decides what to do based on project state
  Runs on demand (human or REFACTOR entry from dream)
  Produces: docs/INDEX.md + draft docs for gaps

Story 2: roll-.dream Scan 6  (dream enhancement, US-AUTO-028)
  Continuous — "what has drifted since last check?"
  Runs nightly; produces REFACTOR entries → loop → $roll-doc fills them
```

## Story 1 — `roll-doc` skill

**Input**: project root (works on any project roll is invoked on)

**Four phases, always run in sequence:**

```
Phase 1 — Scan & Index
  Read all *.md + convention files → produce/update docs/INDEX.md
  Classify: guide / domain / module / convention / stray
  Convention markers: AGENTS.md, CLAUDE.md, CONVENTIONS.md, CONTRIBUTING.md,
                      ARCHITECTURE.md, ADR-*.md

Phase 2 — Gap Analysis
  Compare index against codebase structure
  Flag: dirs with ≥3 source files and no linked doc

Phase 3 — Fill (only when gaps found)
  For each gap: read ≤20 source files, infer module purpose / API / deps / config
  Generate draft at conventional location:
    module gap    → <dir>/README.md
    no docs/domain/ → docs/domain/context-map.md (draft)
    no conventions  → docs/CONVENTIONS.md (draft)
  All drafts: "> Draft — auto-generated by roll-doc on YYYY-MM-DD. Review before treating as authoritative."
  Skips existing files; --force to overwrite

Phase 4 — Report
  N docs indexed, N gaps found, N drafts generated
  Recommend: which drafts to review first (by staleness / module size)
```

**Output**: `docs/INDEX.md` (always updated) + draft files (only for gaps)

**Idempotent**: re-running updates INDEX.md in place; no duplicate drafts.

## Story 2 — roll-.dream Scan 6

**When**: nightly, after existing Scans 1–5

**Checks**:

A. **Stale docs** — `docs/INDEX.md` entry whose owner code was committed more recently than the doc itself (>30 day lag, and doc contains specific file references)

B. **Undocumented conventions** — recurring patterns in code (≥5 occurrences) with zero mentions in any `.md`:
   - ENV vars (`process.env.X`, `os.getenv("X")`)
   - Repeated TODO/FIXME/HACK comment clusters
   - Identical file structure across ≥3 modules (implicit template)

C. **Architecture doc drift** — module directories (≥3 source files) absent from `docs/domain/*.md`

**Output**: REFACTOR entries → loop → executes with `$roll-doc-fill` (once skill exists) or `$roll-build`

## Dependency Ordering

```
US-SKILL-008 (roll-doc)
    │
    ▼
US-AUTO-028  (dream Scan 6)    depends-on: US-SKILL-008
  (can ship without roll-doc, but REFACTOR entries fall back to $roll-build)
```

## Non-Goals

- Not a documentation generation AI that writes authoritative content — only drafts
- Not a linter for doc formatting — only tracks existence and staleness
- Not replacing manual doc review — the human still approves before merging
