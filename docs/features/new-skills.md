## US-SKILL-008 `roll-doc` — Legacy 项目文档自动化（索引 + 补全一体） 📋

**Created**: 2026-05-12
**Plan**: [legacy-doc-automation-plan.md](legacy-doc-automation-plan.md)

- As a developer working on a legacy project with scattered or missing documentation
- I want a single `$roll-doc` skill that analyzes the project and decides what to do
- So that I don't need to know which sub-tool to call — it reads the situation and acts

**Domain Model:**
- Context: Skill Ecosystem
- Aggregate: SkillRegistry owns [DocSkill]
- Events raised: [DocIndexGenerated], [DocDraftGenerated] → consumed by roll-.dream (Scan 6), loop

**Behavior (project-driven, no manual mode switching):**

The skill reads the project first, then decides:

```
Phase 1 — Scan & Index
  Read all *.md + convention files → produce/update docs/INDEX.md
  Classify: guide / domain / module / convention / stray

Phase 2 — Gap Analysis
  Compare index against codebase structure
  Identify: missing module READMEs, missing architecture docs, undocumented conventions

Phase 3 — Fill (only if gaps found)
  For each gap: read source files, generate draft at conventional location
  Draft header: "> Draft — auto-generated by roll-doc on YYYY-MM-DD. Review before treating as authoritative."

Phase 4 — Report
  Summary: N docs indexed, N gaps found, N drafts generated
  Recommend: which drafts to review first (by staleness / module size)
```

**AC:**
- [ ] `$roll-doc` with no args runs all four phases on the current project root
- [ ] Scan exclusions: `node_modules/`, `.git/`, `dist/`, `build/`, `.shared/`, `docs/dream/`, `docs/briefs/`
- [ ] Phase 1 scans all `*.md` files (with exclusions) and convention files by name: `AGENTS.md`, `CLAUDE.md`, `CONVENTIONS.md`, `CONTRIBUTING.md`, `ARCHITECTURE.md`, `ADR-*.md`
- [ ] Phase 1 produces/updates `docs/INDEX.md`: table (path / title / category / last modified) + Coverage Summary + Gap Report
- [ ] Phase 2 flags directories with ≥3 source files and no linked doc; threshold stored in skill config, default 3 (adjustable after noise review)
- [ ] Phase 3 generates drafts only for gaps; skips existing files unless `--force` passed — re-run without `--force` and no new gaps is a **no-op** (existing drafts never modified)
- [ ] `--dry-run` flag: executes Phases 1–2, prints Phase 3 output to stdout, writes nothing
- [ ] Phase 3 draft locations: module gap → `<dir>/README.md`, no `docs/domain/` → `docs/domain/context-map.md`, no conventions doc → `docs/CONVENTIONS.md`
- [ ] Phase 3 draft header: `> **Draft** — auto-generated by roll-doc on YYYY-MM-DD. Review before treating as authoritative.`
- [ ] Works on any project (not roll-specific)
- [ ] `skills/roll-doc/SKILL.md` created with When to Use / When Not to Use

**Files:**
