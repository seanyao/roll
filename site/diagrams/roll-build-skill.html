roll-build One entry. Any input. Full delivery. · 一个入口，任意输入，完整交付。

Take one request — whether a US id or a single vague sentence — down a reversible, test-guarded, evidence-landing pipeline to merged-on-main. Every micro-step leaves a green, revertible state; nothing is "done" without fresh evidence. 把一条需求——无论是 US 编号还是一句模糊的话——经一条可逆、测试守护、证据落地的流水线送到合入主干。每个微步都留绿色可回退态；没有新鲜证据就不算完成。

A Story modeStory 模式

Input is US-XXX — read backlog → split Actions → TCR. A planned story executes directly. 输入是 US-XXX —— 读 backlog → 拆 Action → 进 TCR。已规划好的故事直接执行。

B Fly modeFly 模式

Input is one sentence — clarify → design → create the US, then switch to Story mode to execute. 输入是一句话 —— 先 clarify → design → 创建 US，再切回 Story 模式执行。

PHASE 7

Three parallel reviews on the full story diff: reuse (re-implementing existing helpers?) / quality (param sprawl, copy-paste, deep nesting) / efficiency (redundant compute, N+1, hot-path bloat). 对整故事 diff 并行三个评审：复用（有没有重造轮子）/ 质量（参数蔓延、复制粘贴、深嵌套）/ 效率（冗余计算、N+1、热路径膨胀）。

Core philosophy核心理念

  • Clarity over assumptions — unclear scope → clarify first, don't assume.范围不清先 clarify，不臆测。
  • Just-enough planning — plan to the depth the uncertainty demands.规划到不确定性要求的深度即可。
  • TCR rhythm — test-first, micro-steps, green=commit, red=revert.测试先行、微步、绿则提交、红则回退。
  • Push to GitHub — full implementation + commit + push; code on remote.完整实现 + 提交 + 推送，代码在远端。
  • Stay reversible — every micro-step leaves a clean green repo.每个微步都让仓库停在干净的绿色态。

Hard Rule 0硬规则 Hard Rule 0

  • Worktree-first — write work starts in an isolated worktree, no interference.：写类工作一律先开隔离 worktree，互不干扰。
  • PR-at-end — open a PR for review, never push main directly.：最后开 PR 走评审，绝不直推 main。
  • Three hard gates三道硬闸: peer(3.5) · CI(6) · verification(10.5) — no pass, no go.：同行评审(3.5) · CI(6) · 验证(10.5)，不过不放行。
