roll-build One entry. Any input. Full delivery. · 一个入口，任意输入，完整交付。

把一条需求——无论是 US 编号还是一句模糊的话——经一条可逆、测试守护、证据落地的流水线，送到合入主干。每一步留绿色可回退状态，最后必须有新鲜证据才算完成。

A Story mode

输入是 US-XXX —— 读 backlog → 拆 Action → 进 TCR。已规划好的故事直接执行。

B Fly mode

输入是一句话 —— 先 clarify → design → 创建 US，再切回 Story mode 执行。

规划 · Plan
STEP 0
Pre-flight 自检 est_min

按 est_min 估这轮能不能闭环。太大 → 自降级（roll-design 再拆成子故事，chain_depth+1，连拆 2 次封顶），不硬啃必败的一轮。

STEP 1 · 2
读故事 → 拆 Action 2–5 min/步

读 spec 的 AC / Files / 依赖；拆成最小可交付 Action，先挑最小那块。无占位、可直接执行。

STEP 2.5
并行派发（自动判定） worktree 隔离

Action 间无文件冲突 → 自动开 worktree 并行，子 agent 各自 TCR；完成后逐个审、合主干、跑集成。有冲突则串行。

STEP 3
定义验证

测试矩阵：happy path + 边界/失败/回归；明确本仓"线上验证"是什么（URL / 端点 / 日志信号）。

构建 · Build
PHASE 3.5
同行评审闸 ⛩ GATE 10s 可跳过

跨模块 / 架构决策 / 破坏性操作 / 高风险信号 → 触发跨 agent 评审。AGREE 放行；REFINE/OBJECT 改方案重审。

PHASE 4
测试设计评审

写实现前先审测试：测的是不是对的行为？边界覆盖了吗？测试独立确定吗？——TCR 只保证过测，先确保测试对。

PHASE 5 · 核心
♻ TCR 实现循环 Test && Commit ‖ Revert

微步推进：先写测试（确认 红）→ 最小实现 → 跑测试。每步累积 3–5 个微提交，每个都是有保障的工作态。

写/改测试→ 最小实现→ 跑测试→ 🟢 绿 = commit/ 🔴 红 = revert ↺
PHASE 5.5
E2E 沉淀

为这故事的黄金路径存一个 E2E 用例，沿用本仓既有 E2E 设施。每个沉淀的用例成为可重放的回归。

交付 · Ship
PHASE 6
推前 CI 本地闸 ⛩ GATE

推送前本地跑全套 CI（format + lint + build + test）。红则修到绿，绝不带红推送。

PHASE 7
推前三轴代码评审 reuse · quality · efficiency

对整故事 diff 并行三个评审：复用（有没有重造轮子）/ 质量（参数蔓延、复制粘贴、深嵌套）/ 效率（冗余计算、N+1、热路径膨胀）。

PHASE 8 · 9
提交推送 → 盯 CI → 部署

worktree-first 开分支、PR、绿了合主干（绝不直推 main）。CI 红则诊断 → 新 TCR 微步修 → 重推。

验证 · Verify
PHASE 10
运行时验证

web 在部署 URL 验、CLI 真跑命令验、库用示例脚本验。happy path + 边界 + 无回归。

PHASE 10.5
验证闸 ⛩ MANDATORY

翻 Done 前必须本会话新鲜跑出的证据：测试过、构建过、线上验证、无回归。"我确认过了"不算证据。

PHASE 10.6
验收证据落地 ac-map · attest · 截屏

每条 AC 标 pass/partial/… + 引真实证据 → roll attest 出报告 → roll index 刷档案。改前/改后成对截屏是最强证据。

核心理念 · Core Philosophy

  • Clarity over assumptions — 范围不清先 clarify，不臆测。
  • Just-enough planning — 规划到不确定性要求的深度即可。
  • TCR rhythm — 测试先行、微步、绿则提交、红则回退。
  • Push to GitHub — 完整实现 + 提交 + 推送，代码在远端。
  • Stay reversible — 每个微步都让仓库停在干净的绿色态。

硬规则 · Hard Rule 0

  • Worktree-first：写类工作一律先开隔离 worktree，互不干扰。
  • PR-at-end：最后开 PR 走评审，绝不直推 main。
  • 三道硬闸：同行评审(3.5) · CI(6) · 验证(10.5)，不过不放行。
