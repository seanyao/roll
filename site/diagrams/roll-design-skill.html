roll skill · design before coderoll 技能 · 先设计再编码

roll-design Discuss. Model. Decompose. No code. · 讨论、建模、拆解，不写代码。

Turn an idea or a vague sentence into INVEST stories with a concrete, signed-off design behind them. DDD modeling depth scales with risk; decomposition only ever slices an agreed design — it never substitutes for one. The skill writes backlog and spec artifacts and stops before implementation. 把一个想法或一句模糊的话，变成背后有具体、已签字方案的 INVEST 故事。DDD 建模深度随风险伸缩；拆解永远只是切分一个已认可的方案，绝不替代设计本身。本技能只写 backlog 与 spec 产物，到实现前即止步。

A Interactive mode交互模式

Input is vague or an approach is open — run Clarify (3–5 questions) then Discuss (opinionated, multi-turn) before designing. 输入模糊或方向开放 —— 先 Clarify（问 3–5 个问题）再 Discuss（有主张、多轮），然后才设计。

B Non-interactive mode非交互模式

--from-file / --from-idea / high-confidence input — skip Clarify & Discuss, write stories straight to backlog as 📋 Todo. --from-file / --from-idea / 高置信输入 —— 跳过 Clarify 与 Discuss，故事直接以 📋 Todo 写入 backlog。

Core philosophy核心理念

  • Clarity over assumptions — unclear scope → clarify first, never assume.范围不清先 clarify，绝不臆测。
  • Design before decompose — a concrete artifact comes first; splitting only slices it.先有具体方案产物，拆解只是切分它。
  • Show a worked sample — no complete worked sample → the design is not done.拿不出完整样例 → 设计就不算完成。
  • Depth on a dial — DDD depth scales with risk: full / slice / tag.DDD 深度随风险伸缩：full / slice / tag。
  • Cycle-grained stories — one story ≈ one AI cycle (5–10 min); too big → split.一个故事 ≈ 一个 AI cycle（5–10 min）；太大就拆。

Hard Rules硬规则

  • No implementation不实现 — this skill writes backlog/spec only; it never starts coding (use roll-build).：本技能只写 backlog/spec，绝不开始编码（写代码用 roll-build）。
  • Design-gated decompose设计闸守拆解: no story split until a detailed design exists and the owner signs off.：没有详细方案且 owner 未签字，不得拆故事。
  • Doc-refresh closer文档收尾: any user-visible change ⇒ a closing doc-refresh story, EN/ZH on separate lines.：任何用户可见变更 ⇒ 末尾追加文档刷新故事，中英分行。
  • Backlog ↔ spec consistencybacklog ↔ spec 一致: index rows and spec files must stay in sync; mint via roll story new.：索引行与 spec 文件必须同步；一律用 roll story new 铸造。
