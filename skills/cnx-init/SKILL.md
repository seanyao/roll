# Project Init

> Architecture Constraints, Development Discipline, and Engineering Common Sense are defined in the project AGENTS.md (generated by `cybernetix init`). This skill focuses on scaffolding.

Initialize a project skeleton that supports the CNX workflow.

## When to Use

- Starting a new project that uses the CNX workflow
- Need to generate `BACKLOG.md`, `docs/plans/`, `AGENTS.md`, CI/CD, and Sentinel infrastructure

## What It Sets Up

- `BACKLOG.md` as the core workspace
- `docs/plans/` as the design proposal directory
- `AGENTS.md` as the constraints and routing entry point
- `.github/workflows/` as the CI/CD and patrol entry point

## Workflow

```
Design → Build → Check → Fix → Continuous cycle

Design:  $cnx-design      → docs/features/ + BACKLOG.md
Build:   $cnx-story-build → TCR → CI/CD → Deploy
Check:   $cnx-sentinel    → Scheduled patrol → Discover issues
Fix:     $cnx-fix-build   → TCR → Deploy → Verify
```

## Usage Flow

### 1. Start New Project

```bash
$cnx-init
# Answer a few questions...
# ✅ Project created successfully
```

### 2. Plan Phase

```bash
# Design a new feature
$cnx-design "user system design"

# Output:
# - docs/plans/2024-01-20-user-system/design.md
# - BACKLOG.md updated with:
#   * US-001 User registration
#   * US-002 User login
#   * US-003 Password reset
```

### 3. Do Phase

```bash
# Develop a Story
$cnx-story-build US-001

# Flow:
# 1. Read BACKLOG.md US-001
# 2. TCR development
# 3. CI/CD Deploy
# 4. Update BACKLOG.md: US-001 ✅
```

### 4. Check Phase (Automated)

```yaml
# .github/workflows/sentinel.yml runs automatically
name: Sentinel Patrol
on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours

jobs:
  patrol:
    steps:
      - run: $cnx-sentinel patrol --mode=normal
      # Randomly spot-check 10 ✅ Stories
      # Generate report
      # If issues found, create GitHub Issue
```

### 5. Act Phase (if needed)

```bash
# If Sentinel finds FIX-001
$cnx-fix-build FIX-001

# TCR fix → Deploy → Sentinel verify
```

## Quick Commands Reference

| Phase | Command | Purpose |
|-------|---------|---------|
| Plan | `$cnx-design "requirement"` | Plan design, split into Stories |
| Do | `$cnx-story-build US-XXX` | Develop a Story |
| Do | `$cnx-fix-build FIX-XXX` | Fix a Bug |
| Do | `$cnx-roll-build "one-liner"` | Quick implementation |
| Check | `$cnx-sentinel patrol` | Scheduled patrol |
| Check | `$cnx-bb-debug URL` | Deep diagnosis |
| Act | (automatic) | Create FIX, return to Do |

## Best Practices

1. **BACKLOG is the core** - All work must be tracked in BACKLOG
2. **Plan before building** - Design complex features in docs/plans/ before development
3. **TCR development** - All code changes use Test && Commit || Revert
4. **Sentinel watches over** - Automatic patrols after release, discovering issues
5. **Continuous improvement** - Issue-driven, constantly optimizing
