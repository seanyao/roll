# BB Analyzer

Analyze diagnostic reports generated by `wk-bb-debug`.

## When to Use

- User uploads a diagnostic file (`diagnostics-*.json`, `bb-report.json`)
- User says: "analyze BB data", "look at the diagnostic file"
- Debugging page issues (blank page, errors, broken functionality)

## Supported Report Formats

| Format | Source | Description |
|--------|--------|-------------|
| Native BB | Page with Black Box | `window.__BB_DATA__` or `localStorage.bb_diagnostic` |
| Universal | wk-bb-debug collector | Playwright injected collector data |
| Legacy | Old diagnostic files | Backward compatible |

## Quick Start

```bash
# Analyze an existing report
$wk-bb-analyzer /tmp/bb-report.json

# Or collect first, then analyze
$wk-bb-debug https://example.com/page
$wk-bb-analyzer /tmp/bb-report.json
```

## Analysis Process

### Step 1: Load Report

```javascript
const report = JSON.parse(fileContent);
const mode = report.mode; // 'native-bb' | 'universal'
```

### Step 2: Analyze Based on Mode

#### Native BB Mode Fields

```javascript
const bbData = report.diagnostic.bbData;

// Content state
bbData.contentState?.hasText
bbData.contentState?.contentLength

// Audio state
bbData.audioState?.src
bbData.audioState?.error
bbData.hasAudio

// Errors
bbData.errors
```

#### Universal Mode Fields

```javascript
const d = report.diagnostic;

// Console
d.console.errors
d.console.warnings

// Network
d.network.failed
d.network.slow

// DOM
d.dom.title
d.dom['#root']
d.dom.htmlLength

// Performance
d.performance.loadComplete
d.performance.domContentLoaded

// Errors
d.errors
```

### Step 3: Generate Report

```markdown
## 📊 BB Diagnostic Analysis Report

### 📋 Basic Info
| Field | Value |
|-------|-------|
| Diagnostic Mode | {native-bb / universal} |
| Page URL | {url} |

### 🔴 Key Findings
| Metric | Value | Status |
|--------|-------|--------|
| Console Errors | {N} | {🔴>0 ✅0} |
| Network Failed | {N} | {🔴>0 ✅0} |

{Root cause}

### 🔧 Suggested Fix
{Fix suggestions}
```

## Common Issues

### Blank Page
- `dom.htmlLength < 500`
- `dom['#root'].visible = false`
- **Fix**: Check console errors, add error boundary

### Content Not Loading
- `hasText = false` / `contentLength = 0`
- **Fix**: Check API, refresh OSS URL, fix useEffect

### Audio Error
- `audioState.error` exists
- **Fix**: Refresh signed URL, check audio format

### Network Failure
- `network.failed` has 4xx/5xx
- **Fix**: Check API routes, add CORS

## No Report File?

If the user has no diagnostic file:

> No diagnostic file found. Run `$wk-bb-debug` first to collect data:
>
> ```bash
> $wk-bb-debug https://your-site.com/page
> ```
>
> Then I will automatically analyze the generated report.
